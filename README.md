# STAR LEDGER ✦ 追星總帳

手機版追星記帳網頁：周邊訂單、待到貨、代購分帳、現貨販售、活動場次與收支總帳，資料存在你自己的 Google 試算表。

- 純靜態網頁（HTML/CSS/JS），部署在 GitHub Pages
- 五種主題色（奶茶／抹茶／海鹽／櫻花／夜幕），設定頁隨時切換
- 離線可用：資料快取在手機瀏覽器，連線後再同步

## 功能

| 頁面 | 內容 |
| --- | --- |
| 訂單 | 訂單與多品項編輯、原幣＋台幣雙金額、匯率回算、國際運費估算、整筆複製 |
| 待到貨 | 依品項彙總待到貨數量，展開看來源訂單，一鍵標記到貨 |
| 代購對象 | 依對象彙整品項與應收金額，逐項勾選付清 |
| 販售 | 品項歸屬設「現貨」自動建立販售紀錄，成本快照＋重算、售價與已售數量 |
| 活動 | 演唱會／簽售場次紀錄，可從活動直接新增票券、交通、住宿流水 |
| 總帳 | 依活動彙整支出、搜尋與分類／結清篩選、分帳應結已結 |
| 設定 | 主題色、雲端連線、JSON 備份匯出匯入、全部上傳 |

## 第一次使用：連接 Google 試算表（約 3 分鐘）

網頁「讀＋寫」試算表需要透過 Google Apps Script（免費），步驟：

1. 開啟你的 Google 試算表 → 上方選單「擴充功能」→「Apps Script」。
2. 刪掉編輯器裡的預設內容，把本專案 [`apps-script/Code.gs`](apps-script/Code.gs) 的全部內容貼上，存檔。
3. 右上「部署」→「新增部署作業」→ 類型選「網頁應用程式」：
   - 執行身分：**我**
   - 誰可以存取：**知道連結的任何人**
4. 按「部署」並授權（第一次會跳出權限確認，選自己的帳號 → 進階 → 前往專案）。
5. 複製產生的「網頁應用程式網址」（`https://script.google.com/macros/s/…/exec`）。
6. 開啟本 App → 「設定」→ 貼到「Apps Script 網址」→「儲存並測試連線」。

連線成功後，工作表（orders / items / sales / events / ledger）會自動建立。

### 匯入舊資料

設定頁 →「匯入 JSON 備份」選擇舊版匯出的備份檔 → 依提示「全部上傳到試算表」。

### 加一層密碼（選用）

`Code.gs` 第一行 `SHARED_KEY` 填一組自訂字串並重新部署，App 設定頁「金鑰」填相同的值。

## 本機開發

```bash
python3 -m http.server 8642
# 打開 http://localhost:8642
```

## 隱私

- 這個 repo 只有程式碼，沒有任何個人資料。
- 你的資料只存在：你的 Google 試算表＋你手機瀏覽器的快取。
- Apps Script 網址等連線設定存在瀏覽器 localStorage，不會進到 repo。
